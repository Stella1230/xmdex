# Dashboard 角色差异化设计

## 目标

根据用户角色（超级管理员 / 部门负责人 / 小组长 / 普通员工）渲染不同的首页看板 UI，数据自动隔离，无手动越权入口。

## 角色识别

- 从 `localStorage.getItem('roleId')` 读取
- Login.jsx 登录时额外存储 `roleId`（`user.roleId`）
- 角色 ID 映射：`ROLE_ADMIN` / `ROLE_DEPT_MANAGER` / `ROLE_GROUP_LEADER` / `ROLE_EMPLOYEE`

## 架构

```
Dashboard.jsx（路由入口，读取角色，组合组件）
├── DashboardHeader.jsx    头部欢迎语 + 角色 Tag + 时间筛选 + 刷新按钮
├── StatCards.jsx          指标卡（根据角色显示 3~5 个）
├── ChartSection.jsx       图表区（根据角色显示 1~4 张）
└── RecentTaskTable.jsx    最近待办列表（根据角色限制操作列）
```

## 各角色 UI 规格

### 超级管理员（ROLE_ADMIN）

| 区域 | 内容 |
|------|------|
| 头部 | 欢迎语 + `<Tag color="red">超级管理员</Tag>` + 时间范围筛选（周/月） + 刷新按钮 |
| 指标卡（5个） | 项目总数、待办总数量、进行中、已逾期、已完成；每个下方小字「较昨日 +X」 |
| 图表（4张） | 饼图：全公司状态分布；柱状图：各部门待办对比；环形图：项目平均完成度；折线图：近7天新增/完成趋势 |
| 明细列表 | 全公司最近10条待办；操作：查看、编辑、删除 |
| 空状态 | 「暂无待办数据，请先创建项目和任务」 |

### 部门负责人（ROLE_DEPT_MANAGER）

| 区域 | 内容 |
|------|------|
| 头部 | 欢迎语 + `<Tag color="blue">部门负责人</Tag>` + 刷新按钮 |
| 指标卡（5个） | 本部门项目数、部门待办总数、进行中、已逾期、部门整体完成率 |
| 图表（4张） | 饼图：本部门状态分布；柱状图：部门内小组待办对比；环形图：部门项目完成度；折线图：部门近7天完成趋势 |
| 明细列表 | 本部门最近待办；操作：查看、编辑、调整负责人 |
| 空状态 | 「暂无待办数据」 |

### 小组长（ROLE_GROUP_LEADER）

| 区域 | 内容 |
|------|------|
| 头部 | 欢迎语 + `<Tag color="green">小组长</Tag>` + 刷新按钮 |
| 指标卡（4个） | 小组待办总数、进行中、已逾期、已完成 |
| 图表（2张） | 饼图：小组状态占比；柱状图：组内成员待办对比 |
| 明细列表 | 直属成员待办；操作：查看、标记状态 |
| 空状态 | 「暂无待办数据」 |

### 普通员工（ROLE_EMPLOYEE）

| 区域 | 内容 |
|------|------|
| 头部 | 欢迎语 + `<Tag color="default">普通员工</Tag>` + 刷新按钮 |
| 指标卡（3个） | 我的待办总数、进行中、已完成 |
| 图表（1~2张） | 饼图：个人状态分布；折线图（可选）：个人近7天完成趋势 |
| 明细列表 | 个人待办；操作：查看、标记完成/取消完成 |
| 空状态 | 「暂无分配给你的待办」 |

## 交互细节

1. **数据隔离**：前端不提供部门/人员筛选器，数据由后端根据角色返回
2. **跳转联动**：
   - 点击指标卡 → 跳转待办管理页，带入状态筛选
   - 点击柱状图柱子 → 跳转对应筛选后的待办列表
   - 点击待办标题 → 打开待办详情弹窗
3. **时间范围筛选**：仅超级管理员可见，按周/月切换
4. **刷新按钮**：右上角，手动重新拉取统计数据

## 后端接口

| 接口 | 用途 | 备注 |
|------|------|------|
| `getTodoBoardData` | 统计数据（指标卡+图表） | 后端根据角色返回对应范围数据 |
| `getTodoTaskList` | 最近待办列表 | 后端根据角色过滤 |
